Privacy policy for TomatenKuchen

Website

External providers

  • This website uses Cloudflare to host the website. The service has its HQ in the US and may transfer personal related data (PII) there. For details, see their privacy policy.
  • Discord is used to display Discord resources, mostly avatars and server icons. It is also used when logging in with Discord.
  • We use Simple Analytics to be able to collect anonymous usage statistics of the website. You can learn more about the collected data on their "What we collect" page.

Local data store

  • token: A token to authentificate a user on the site.
  • cookie-dismiss: Stores the consent to not show the consent choice again.
  • theme, optional: Stores the color scheme selected in the sidebar.
  • lang, optional: Stores the language selected in the sidebar.
  • user, optional: Stores the current global Discord name.
  • avatar, optional: Is used to display the current user avatar in the top right corner.

Data collected

In short: Only necessary data the user ("you") provides to us and is necessary to provide the functionality of our services is stored. We only use it for the operation of the bot and the website and never share it with third parties unless you (implicitely) allow us to do so.


  • User IDs: Blacklist, reminders, economy, todo, AFK, birthday, user settings, modlogs, rejoin roles, Bot badges, levels, antiraid², antispam², server statistics³, polls, private voice channels, forms, customroles, partnerships
  • User avatars: Tickets¹
  • Server IDs: Server statistics, server configuration, modlogs, levels, active invites (invite tracking), polls, forms, partnerships
  • Channel IDs: Server configuration, Minecraft server status, tickets, polls, private voice channels, partnerships
  • Role IDs: Server configuration, rejoin roles, customroles
  • Message IDs: Server configuration, Minecraft server status, starboard, suggestions, giveaways, autodelete, partnerships
  • Message content: Antispam² (detecting repeated messages), antiscam (only if links/invites in content: using two external services, no saving), suggestions and partnerships (only if the message was sent in a channel selected for them)
  • Message mentions: AFK
  • Attachments: Tickets¹, suggestions and starboard (images only)
  • Commands used: Bot and user statistics, bug fixing
  • Form responses
  • Entered Bot data of custom bots
  • Depending on which server parts are selected, backups may store all Discord server related data.
  • All data stored in the database is encrypted with a recognized encryption method.
  • To ensure data availability, backups are created several times a day on external storage media.
¹Since attachments and user avatars are deleted by Discord after some time but may be important for context in ticket transcripts, they are stored on the TomatenKuchen server. You can disable this in your user settings.
²The data collected by the antiraid and antispam systems isn't stored permanently and is deleted within a few minutes after collection.
³The collected active chatters user IDs are converted at 00:00 CET/CEST to an amount and the list of IDs is deleted.

You can view your user data on the dataexport page.

Who has access to this data?

The bot developer, devtomatocake, has access to the data and can modify or delete it at any time.
Additionally some contributors have access to server logs.
The bot and the database are hosted on a server in Germany rented from Hetzner.

  • Server settings, integrations, custom commands, reaction, button and select roles, server statistics, form responses: Once the bot is removed from a server, a four day timer starts at which end the data gets deleted. You can also request it to be deleted immediately.
  • Reminders: Reminders are automatically deleted once the reminder has reminded you.
  • AFK: Your AFK status is deleted once you send a message in a channel the bot can see.
  • Most data can be deleted the way it was created.

If you have other wishes or want all³ of your data deleted, you can create a ticket on our support server.

After creating a data deletion ticket, your data will usually be deleted within a few hours.

³Excepted are: Modlogs and form responses without server staff approval, blacklist
