Privacy policy for TomatenKuchen

What information is collected and why?

In short: Only necessary data the user ("you") provides to us is stored.

Website:

This site uses Cloudflare for faster page loading. The service has their HQ in the US, which means that your data may be processed there. The data is only used anonymously.
Additionally, this site uses cookies to store required information, e.g. login data and your cookie choice. Optional cookies include the user avatar, language and theme choice. You aren't tracked identifyable by us!
If the avatar is stored a request to Discord (HQ also in US) is made on page load to display the avatar.

Discord bot:

  • User IDs: Blacklist, reminders, economy, todo, AFK, birthday, user settings, modlogs, rejoin roles, tk badges, levels, antiraid², antispam², server statistics³, polls, private voice channels (temporary), customroles, forms
  • User avatars: Tickets¹
  • Server IDs: Server statistics, server settings, modlogs, levels, reaction, integrations, button and select roles, custom commands, polls, forms
  • Channel IDs: Server settings, Minecraft server status, tickets, polls, private voice channels (temporary)
  • Role IDs: Server settings, reaction, button and select roles, rejoin roles, customroles
  • Message IDs: Reaction, button and select roles, Minecraft server status, starboard, suggestions, giveaways, autodelete
  • Message content: Antispam² (detecting repeated messages), antiscam (only if links/invites in content using two external services, no saving)
  • Message mentions: AFK
  • Attachments: Tickets¹, suggestions & starboard (images only)
  • Commands used: Bot and user statistics, bug tracking
  • Form responses: Forms
  • Bot data of custombots: Name, token, users with team access
  • Depending on which server parts are selected by the server admin, backups may store all server related data.
  • At least once a day, a backup is stored externally (developer PC and Hetzner storage in Helsinki - both with high security standards)
¹Since attachments and user avatars are deleted by Discord after some time but may be important for context in ticket transcripts, they are stored on the TomatenKuchen server. You can disable this in your user settings.
²The data collected by the antiraid and antispam systems isn't stored permanently and is deleted within a few minutes after collection.
³The collected active chatters user IDs are converted at 00:00 CET/CEST to an amount and the list of IDs is deleted.

You can view your user data on the dataexport page.

Who has access to this data?

The bot developer, devtomatocake, has access to the data and can modify or delete it at any time.
Additionally some contributors have access to server logs.
The bot and the database are hosted on a server in Germany provided by Hetzner.

  • Server settings, integrations, custom commands, reaction, button and select roles, server statistics, form responses: Once the bot is removed from a server, a four day timer starts at which end the data gets deleted. You can also request it to be deleted immediately.
  • Reminders: Reminders are automatically deleted once the reminder has reminded you.
  • AFK: Your AFK status is deleted once you send a message in a channel the bot can see.
  • Most data can be deleted the way it was created.

If you have other wishes or want all³ of your data deleted, you can create a ticket on our support server.

After creating a data deletion ticket, your data will usually be deleted within a few hours.

³Excepted are: Modlogs and form responses without server staff approval, blacklist

Datenschutzerklärung für TomatenKuchen

Welche Daten werden gespeichert und wozu werden sie benötigt?

Kurz gesagt: Es werden nur notwendige Daten gespeichert, die der Nutzer ("du") uns durch die Nutzung unserer Dienste zur Verfügung stellt.

Website:

Diese Website nutzt Cloudflare zur schnellen Bereitstellung der Seite unabhängig vom Standort des Nutzers. Der Dienst hat seinen Hauptsitz in der USA und überträgt möglicherweise personenbezogene Daten, in diesem Fall IP-Adressen, zu Logzwecken dorthin. Diese Daten werden nur anonymisiert verarbeitet, zu Einzelheiten siehe dessen Datenschutzerklärung.
Zusätzlich werden auf deinem Endgerät Cookies gespeichert, die für die Funktion der Website notwendig sind, dazu gehören Anmeldeinformationen und die Cookie-Auswahl. Wenn optionale Cookies aktiviert sind werden zusätzlich weitere Daten gespeichert, z. B. den Avatar des Nutzers, die ausgewählte Sprache und das Theme. Du wirst niemals personenbezogen von uns getrackt!
Wenn der Avatar gespeichert wird wird beim Laden der Seite eine Anfrage an Discord durchgeführt, um den Avatar anzeigen zu können. Discord sitzt ebenfalls in den USA.

Discord-Bot:

  • Nutzer-IDs: Blacklist, Reminder, Economy, Todo, AFK, Geburtstag, Nutzereinstellungen, Modlogs, Rejoin-Rollen, TK-Badges, Level, Antiraid², Antispam², Serverstatistiken³, private Sprachkanäle (temporär), Umfragen, Formulare, Customrollen
  • Nutzer-Avatar: Ticketsystem¹
  • Server-IDs: Serverstatistiken, Servereinstellungen, Modlogs, Level, Integrationen, Reaction-, Button- und Selectroles, aktive Einladungen, Umfragen, Fehlerlogs, Formulare, Sync
  • Kanal-IDs: Servereinstellungen, Minecraft-Serverstatus, Ticketsystem, Umfragen, private Sprachkanäle (temporär)
  • Rollen-IDs: Servereinstellungen, Reaction-, Button- und Selectroles, Rejoin-Rollen, Customrollen
  • Nachrichten-IDs: Reaction-, Button- und Selectroles, Minecraft-Serverstatus, Starboard, Vorschläge, Giveaways, Autodelete
  • Nachrichteninhalt: Antispam² (Erkennung wiederholter Nachrichten), Antiscam (nur wenn Links/Einladungen enthalten Überprüfung durch zwei externe Dienste, keine Datenspeicherung)
  • Nachrichten-Erwähnungen: AFK
  • Anhänge: Ticketsystem¹, Vorschläge & Starboard (nur Bilder)
  • Verwendete Befehle: Bot- und Nutzerstatistiken, Fehlerbehebung
  • Formularantworten: Formulare
  • Botdaten von Custombots: Name, Token, Mitglieder mit Teamzugriff
  • Je nachdem, welche Serverteile von dem Serveradmin für ein Backup ausgewählt worden sind, speichern Backups alle mit dem Server verbundenen Inhalte.
  • Mindestens einmal pro Tag wird ein externes Backup erstellt (Entwickler-PC und Hetzner-Backupspeicher in Helsinki - beide mit hohen Sicherheitsstandards)
¹Da Anhänge und Nutzer-Avatare automatisch von Discord nach einer gewissen Zeit gelöscht werden, aber für den Kontext in Ticket-Transcripten wichtig sein können, werden diese auf dem TomatenKuchen-Server gespeichert. Diese Funktion kann in den Nutzereinstellungen deaktiviert werden.
²Die von Antiraid bzw. Antispam gesammelten Daten werden nicht permanent gespeichert bzw. innerhalb weniger Minuten nach der Erfassung gelöscht.
³Die gesammelten aktiven-Chatter-Nutzer-IDs werden täglich um 0 Uhr deutscher Zeit in die Anzahl der Nutzer umgewandelt und danach gelöscht, die einzelnen IDs sind ab dann anonym und davor nicht öffentlich.

Du kannst deine von TomatenKuchen gespeicherten Daten auf der Dataexport-Seite online einsehen.

Wer hat Zugang zu den Daten?

Der Bot-Entwickler, devtomatocake, hat Zugriff auf die Daten und kann sie jederzeit ändern oder löschen.
Zusätzlich haben einige Contributors Zugriff auf Serverlogs.
Der Bot und die Datenbank werden auf einem von Hetzner gemieteten Server mit Standort in Deutschland gehostet.

  • Servereinstellungen, Serverstatistiken, Integrationen, Reaction-, Button- und Selectroles, Formularantworten: Sobald der Bot vom Server entfernt wird, wird ein vier Tage langer Timer gestartet, bei dessen Ablauf alle Daten gelöscht werden. Du kannst auch das sofortige Löschen der Daten beantragen.
  • Reminder: Reminder werden nach Ablauf direkt gelöscht.
  • AFK: Dein AFK-Status wird wieder gelöscht, sobald du eine Nachricht in einem Kanal sendest, den der Bot sehen kann.
  • Nahezu alle Daten lassen sich auf dem Weg löschen, auf dem sie erstellt worden sind.

Wenn du andere Wünsche hast oder alle³ deine Daten gelöscht werden sollen, kannst du auf unserem Support-Server ein Ticket öffnen.

Nach dem Öffnen eines Tickets werden die Daten in der Regel innerhalb weniger Stunden gelöscht.

³Davon ausgenommen sind: Modlogs und Formularantworten ohne Einwilligung beteiligter Serveradmins, Blacklist
