TomatenKuchen's privacy policy and Terms of Service/Use

By using the site tomatenkuchen.eu, the bot TomatenKuchen, or another bot that links to this site, you agree to it's privacy policy and terms of Service/Use. This page was last updated on 2023-03-13.


Privacy policy for TomatenKuchen

What information is collected and why?

In short, only necessary information that you provide when using our services is stored.

  • User IDs: Blacklist, reminders, economy, todo, AFK, birthday, user settings, modlogs, rejoin roles, badges, levels, polls, antiraid², antispam², statistics³
  • User avatars: Tickets¹
  • Server IDs: Statistics, server settings, modlogs, levels, reaction, button and select roles, custom commands, polls
  • Channel IDs: Server settings, Minecraft server status, tickets, level multipliers, polls
  • Role IDs: Reaction, button and select roles, autorole, rejoin roles
  • Message IDs: Reaction, button and select roles, Minecraft server status, starboard, suggestions, giveaways
  • Message content: Antispam² (detecting repeated messages), antiscam (using two external services, no saving)
  • Message mentions: AFK
  • Attachments: Tickets¹, suggestions (images only)
  • Commands used: Bot and user statistics, bug tracking
¹Since attachments and user avatars are deleted by Discord after some time but may be important for context in ticket transcripts, they are stored in a channel on the TomatenKuchen Discord. You can disable this in your user settings.
²The data collected by the antiraid and antispam systems isn't stored permanently and is deleted within a few minutes after collection.
³The collected IDs are converted at 0 o'clock german time to an amount and the list of IDs is deleted.

You can view your user data on this site.

Who has access to this data?

The bot developer, TomatoCake#2691, has access to the data and can modify or delete it at any time.
Additionally, a few selected, trusted members of the TomatenKuchen Discord have access to server and error logs.
The bot and the database are hosted on a server in Germany provided by Hetzner.

  • Server settings, custom commands, reaction, button and select roles, server statistics: Once the bot is removed from a server, this data is deleted immediately.
  • Reminder, AFK: This data is automatically deleted as soon as the reminder triggered or the AFK state ended.
  • Most data can be deleted the way it was created.

If you have other wishes or want all³ of your data deleted, you can create a ticket on our support server.

After creating a data deletion ticket, your data will usually be deleted within a few hours.

³Excepted are: Modlogs without server staff approval, blacklist

Privacy policy for the Manage Bot

What information is collected and why?

  • User IDs: User settings
  • Server IDs: Custom commands
  • Commands used: Bot statistics, bug tracking

  • Customcommands: Once the bot is removed from a server, this data is immediately deleted.
  • Panels: You can manage your added panels using /settings.

The "Who has access to this data?" section is the same as for TomatenKuchen.

Datenschutzerklärung für TomatenKuchen

Welche Daten werden gespeichert und wozu werden sie benötigt?

Kurz gesagt: Es werden nur notwendige Daten gespeichert, die der Nutzer uns durch die Nutzung unserer Dienste zur Verfügung stellt.

  • Nutzer-IDs: Blacklist, Reminder, Economy, Todo, AFK, Geburtstag, Nutzereinstellungen, Modlogs, Rejoin-Rollen, Badges, Level, Polls, Antiraid², Antispam², Statistiken³
  • Nutzer-Avatar: Ticketsystem¹
  • Server-IDs: Statistiken, Servereinstellungen, Modlogs, Level, Reaction-, Button- und Selectroles, Customcommands, Polls
  • Kanal-IDs: Servereinstellungen, Minecraft-Serverstatus, Ticketsystem, Levelmultiplier, Polls
  • Rollen-IDs: Reaction-, Button- und Selectroles, Autorollen, Rejoin-Rollen
  • Nachrichten-IDs: Reaction-, Button- und Selectroles, Minecraft-Serverstatus, Starboard, Vorschläge, Giveaway
  • Nachrichteninhalt: Antispam² (Erkennung wiederholter Nachrichten), Antiscam (Überprüfung durch zwei externe Dienste, keine Datenspeicherung)
  • Nachrichten-Erwähnungen: AFK
  • Anhänge: Ticketsystem¹, Vorschläge (nur Bilder)
  • Verwendete Befehle: Bot- und Nutzerstatistiken, Fehlerbehebung
¹Da Anhänge und Nutzer-Avatare automatisch von Discord nach einer gewissen Zeit gelöscht werden, aber für den Kontext in Ticket-Transcripten wichtig sein können, werden diese in einem Kanal auf dem TomatenKuchen-Discord gespeichert. Diese Funktion kann in den Nutzereinstellungen deaktiviert werden.
²Die von Antiraid bzw. Antispam gesammelten Daten werden nicht permanent gespeichert bzw. innerhalb weniger Minuten nach der Erfassung gelöscht.
³Die gesammelten IDs werden täglich um 0 Uhr deutscher Zeit in die Anzahl der Nutzer umgewandelt und danach gelöscht, die einzelnen IDs sind ab dann anonym und davor nicht öffentlich.

Du kannst deine Daten auf dieser Seite einsehen. Alternativ kannst du auch den Befehl /misc dataexport verwenden, der jedoch nicht für die Vollständigkeit der Daten garantiert.

Wer hat Zugang zu den Daten?

Der Bot-Entwickler, TomatoCake#2691, hat Zugriff auf die Daten und kann sie jederzeit ändern oder löschen.
Zusätzlich haben ausgewählte, vertrauenswürdige Mitglieder des TomatenKuchen-Discords Zugriff auf Server- und Fehlerlogs.
Der Bot und die Datenbank werden auf einem von Hetzner gemieteten Server mit Standort in Deutschland gehostet.

  • Servereinstellungen, Customcommands, Reaction-, Button- und Selectroles, Serverstatistiken: Sobald der Bot vom Server entfernt wird, werden diese automatisch gelöscht.
  • Reminder, AFK: Diese Einstellungen werden automatisch gelöscht, sobald der Reminder abgelaufen ist oder der AFK-Status beendet wurde.
  • Nahezu alle Daten lassen sich auf dem Weg löschen, wie sie erstellt worden sind.

Wenn du andere Wünsche hast oder alle³ deine Daten gelöscht werden sollen, kannst du auf unserem Support-Server ein Ticket öffnen.

Nach dem Öffnen eines Tickets werden die Daten in der Regel innerhalb weniger Stunden gelöscht.

³Davon ausgenommen sind: Modlogs ohne Einwilligung beteiligter Serveradmins, Blacklist
